# ActionBus 上位机需求文档

---

## 一、概述

上位机作为总线主机（Master），负责向从机发送指令、接收数据、枚举设备功能、以及传输文件。上位机通过串口与总线连接，完整实现 ActionBus v3.1 协议的主机侧逻辑。

---

## 二、串口连接

- 支持选择系统串口设备（如 `/dev/ttyUSB0`、`COM3`）
- 支持配置波特率，预设选项覆盖 1200 / 2400 / 4800 / 9600 / 19200 / 38400 / 57600 / 115200 / 230400 / 460800 / 921600 / 1000000 / 2000000 / 4000000，同时支持手动输入任意波特率值
- 支持配置数据位、停止位、校验位
- 连接状态实时显示（已连接 / 断开）
- 断开后可重连，重连不需要重启上位机

---

## 三、帧收发

- 发送方向严格按照帧格式组包：
  `AA 55 | Addr | FuncCode | Stat | Len | Data | CRC_H CRC_L`
- 接收方向实现字节流状态机解析，正确处理粘包和乱序字节
- 每帧收发均自动计算并校验 CRC-16/MODBUS
- CRC 校验失败的帧静默丢弃，不触发错误弹窗
- 支持原始帧日志（十六进制），可开关

---

## 四、设备发现

上位机连接串口后，可一键执行设备扫描，流程如下：

1. 广播 `Addr=0xFF`，发送 `0xF0 GET_BUS_ADDRESS`，收集所有回复地址
2. 对每个地址逐一查询：
   - `0xF0 GET_PROTOCOL_VERSION` — 确认协议版本兼容
   - `0xF0 GET_DEVICE_ID` — 获取厂商/产品信息
   - `0xF0 GET_ACTION_COUNT` — 获取已注册 Action 数量
3. 对每个设备循环发送 `0xFE GET_ENTRY(index=0..N-1)`，枚举全部功能码和描述
4. 扫描结果以列表形式展示，包含：地址、设备 ID、协议版本、Action 列表

---

## 五、Action 控制

- 从设备发现结果中选择目标设备和功能码
- 支持手动发送 **主机 ON**（`Stat=0x11`）启动 Action
- 支持手动发送 **主机 OFF**（`Stat=0x10`）停止 Action
- 支持在 ON 帧中附加自定义 Data 载荷（十六进制输入）
- 接收从机的响应帧并展示：
  - `Stat=0x01`（从机 ON / 运行中）
  - `Stat=0x00`（从机 OFF / 已完成）
  - `Stat=0x80`（从机错误，显示错误码和描述）

---

## 六、数据展示

- 从机主动上报的数据帧（`Stat=0x01`）实时显示在日志区
- 支持十六进制和 ASCII 双列显示
- 日志可滚动、可清空、可导出为文本文件
- 支持按功能码过滤日志

---

## 七、文件传输

上位机需完整实现文件传输协议（对应从机 `ab_file`）：

### 7.1 操作流程

1. 用户选择目标设备地址和功能码
2. 用户选择本地文件，填写文件类型（`type` 字段，十六进制）
3. 上位机自动执行：
   - 发送 INIT，等待 RSP_READY
   - 按 252 字节/块分割文件，逐块发送 DATA，等待 RSP_ACK
   - 收到 RSP_NAK 时从指定 seq 重发
   - 所有块发完后发送 END（附整文件 CRC-16），等待 RSP_DONE

### 7.2 进度与状态

- 显示传输进度条（已发块数 / 总块数，百分比）
- 显示当前传输速率（字节/秒）
- 显示预计剩余时间
- 传输完成或失败均有明确提示

### 7.3 超时与重试

- 每帧等待从机响应的超时时间可配置（默认 3 秒）
- 超时后自动重发当前帧，最多重试 3 次
- 3 次重试仍失败则发送 ABORT 并提示用户

---

## 八、错误处理

| 情形 | 上位机行为 |
|------|-----------|
| 从机返回 `Stat=0x80` | 显示错误码及对应描述，不自动重试 |
| 帧 CRC 校验失败 | 静默丢弃，在状态栏计数 |
| 等待响应超时 | 可配置重试次数，超出后提示 |
| 串口断开 | 立即提示，停止所有发送 |
| 文件传输中断 | 自动发送 ABORT，释放资源 |

---

## 九、可视化控件系统

参考 VOFA+ 的设计思路，上位机中部为自由画布，用户可从控件库拖入控件并自由排布。

### 9.1 画布与布局

- 中部为自由拖拽画布，控件可任意摆放、缩放、层叠
- 控件库以侧边面板或工具箱形式呈现，拖入即实例化
- 支持保存 / 加载画布布局（JSON 格式）
- 画布支持网格对齐吸附（可开关）

### 9.2 内置控件列表

| 控件 | 功能 |
|------|------|
| **波形图** | 多通道折线波形，支持滚动、缩放、暂停、游标测量 |
| **三维姿态（立方体）** | 根据四元数或欧拉角实时渲染三轴旋转的立方体 |
| **仪表盘** | 指针式圆形仪表，可设量程和告警区 |
| **数字显示** | 单值数字大字显示，支持单位和小数位配置 |
| **进度条** | 横向/纵向进度条，可设最大最小值 |
| **开关按键** | 点击后发送指定 ActionBus 帧（ON/OFF 可分别配置） |
| **滑块** | 拖动产生连续值，实时发送到指定功能码 |
| **文本标签** | 静态或动态文本，可绑定数据通道 |
| **XY 散点图** | 双通道 X-Y 散点/轨迹显示 |
| **状态灯** | 布尔值驱动颜色变化（绿/红/黄） |

### 9.3 JS 脚本解码

每个控件可绑定一段 JavaScript 脚本，脚本负责将收到的帧数据解码为控件可接受的值。

**脚本执行时机：** 每次收到对应功能码的帧时触发。

**脚本上下文（内置对象）：**

```javascript
// 输入：当前帧信息
frame.addr      // 从机地址 (number)
frame.func_code // 功能码 (number)
frame.stat      // Stat 字节 (number)
frame.data      // Data 字节数组 (Uint8Array)
frame.len       // 数据长度 (number)

// 输出：向控件推送数据
channel.push("ch1", value)        // 推送单个值到指定通道
channel.push("x", x, "y", y)     // 推送多个通道
channel.set("label", "文本内容")  // 设置文本

// 工具函数
util.readFloat32BE(data, offset)  // 大端 float32
util.readFloat32LE(data, offset)  // 小端 float32
util.readInt16BE(data, offset)    // 大端 int16
util.readUint32BE(data, offset)   // 大端 uint32
```

**示例：从 Data[0..7] 解析两个 float32 推送到波形图**

```javascript
var ch1 = util.readFloat32BE(frame.data, 0);
var ch2 = util.readFloat32BE(frame.data, 4);
channel.push("温度", ch1);
channel.push("湿度", ch2);
```

**示例：从 Data[0..5] 解析欧拉角推送到姿态控件**

```javascript
var roll  = util.readInt16BE(frame.data, 0) / 100.0;
var pitch = util.readInt16BE(frame.data, 2) / 100.0;
var yaw   = util.readInt16BE(frame.data, 4) / 100.0;
channel.push("roll", roll, "pitch", pitch, "yaw", yaw);
```

- 脚本编辑器内置语法高亮和错误提示
- 脚本运行时错误显示在日志区，不影响其他功能
- 脚本可独立于控件保存为 `.js` 文件，方便复用

### 9.4 数据导出

- 所有通道的历史数据可导出为 **CSV 文件**
- CSV 格式：第一行为列名（时间戳 + 各通道名），后续行为数据
- 支持选择导出时间范围（全部 / 最近 N 秒 / 自定义区间）
- 导出时间戳精度为毫秒
- 波形图控件支持直接右键 → 导出当前可见区间

---

## 十、界面要求

- 分区布局：
  - **左侧**：设备列表（扫描结果）+ 控件库面板
  - **中部**：自由画布（可视化控件区）
  - **右侧**：帧日志区
- 串口参数和设备地址在顶部工具栏常驻
- 所有操作均有操作结果反馈，不出现无响应的按钮
- 支持深色 / 浅色主题

---

## 十一、非功能要求

- 跨平台：支持 Windows 和 Linux
- 启动时间 < 3 秒
- 串口数据处理延迟 < 50ms
- 文件传输效率：在 115200 波特率下接近理论上限（约 10KB/s）
- 波形图在 10 通道、100Hz 刷新率下帧率不低于 30fps
